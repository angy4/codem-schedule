          <h1>Notifications Controller</h1>
          <p>
          Whenever a job enters the <tt>completed</tt> or <tt>failed</tt> state, a
          notification can be sent to either an url or an email address. These can be
          specified using the <tt>notify</tt> parameters in <a
          href="JobsController.html#M000012">Api::JobsController#create</a>.
          </p>
          <p>
          A notification can be <tt>scheduled</tt>, <tt>completed</tt> or
          <tt>failed</tt>.
          </p>
          <p>
          If <tt>scheduled</tt>, the job is still processing, so the notification has
          not been sent yet. If <tt>completed</tt>, the notification has been sent
          successfully. If <tt>failed</tt>, either the url was unreachable or sending
          to the email address returned an error.
          </p>
