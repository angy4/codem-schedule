Notifications Controller

Whenever a job enters the completed or failed state, a notification can be sent to either an url or an email address. These can be specified using the notify parameters in Api::JobsController#create.

A notification can be scheduled, completed or failed.

If scheduled, the job is still processing, so the notification has not been sent yet. If completed, the notification has been sent successfully. If failed, either the url was unreachable or sending to the email address returned an error.
